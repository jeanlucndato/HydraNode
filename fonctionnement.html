Vibeware : Architecture Moderne

Vibeware est un projet de recherche visant à démontrer la supériorité des langages modernes (Rust) et des frameworks asynchrones (FastAPI) pour la création d'implants et de serveurs C2 furtifs.

L'Implant (Agent)

Écrit en Rust pour une gestion mémoire stricte. Utilise l'injection réflexive pour éviter la détection EDR basée sur les fichiers.

Le C2 (Backend)

FastAPI gérant le cycle de vie des beacons. Conçu pour la scalabilité horizontale et l'auto-documentation API.

Fonctionnement : Le Cycle de Vie

Injection & Entropie

L'agent modifie son propre code au repos pour changer sa signature binaire (Polymorphisme).

Beaconing (Pull model)

Aucune connexion entrante. L'agent contacte le serveur périodiquement via HTTPS port 443.

Benchmark Comparatif : Vibeware vs Legacy

Taux de détection (AV/EDR) - Moins c'est mieux Vibeware: 8% vs Legacy: 65%
Consommation RAM (Implant au repos) Rust: 1.2MB vs C++: 4.8MB
Connexions simultanées (C2 Server) FastAPI: 15k/sec vs PHP/Apache: 2k/sec

Analyse Statique

Le binaire Rust généré avec panic='abort' et lto=true produit un exécutable minimaliste de 350 Ko, très difficile à heuristiquer sans signatures spécifiques.

Analyse Réseau

L'utilisation du Jitter (décalage aléatoire) dans le beaconing rend l'analyse de fréquence (FFT) inefficace pour les systèmes de détection d'anomalies.

Stack Technique & Outillage

Rust (Agent)

Crates: `reqwest` (HTTPS), `aes-gcm` (Crypto), `sysinfo` (Recon)

FastAPI (C2)

Uvicorn, Pydantic, SQLAlchemy pour la persistance des tâches.

React (UI)

TailwindCSS, Lucide Icons, WebSockets pour le feedback temps-réel.

Déploiement en Laboratoire

Assurez-vous d'avoir Docker et Rust (Cargo) installés pour le déploiement local.

1. Compilation de l'Agent

cargo build --release --target x86_64-pc-windows-msvc # Stripping symbols for stealth strip target/release/vibeware_agent.exe

2. Lancement du C2 Backend

pip install fastapi uvicorn cryptography uvicorn main:app --ssl-keyfile certs/key.pem --ssl-certfile certs/cert.pem
